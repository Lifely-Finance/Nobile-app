# Contribution Guidelines

Thank you for your interest in contributing to the Nobile App! We welcome contributions from everyone, whether you're fixing bugs, suggesting features, or enhancing documentation.

## Reporting Bugs
- Please report bugs by creating an issue in this repository. Include a clear description of the problem, steps to reproduce, and any relevant screenshots.
- Use the [Bug Report](https://github.com/Lifely-Finance/Nobile-app/issues/new?template=bug_report.md) template for consistency.

## Suggesting Features
- If you have an idea for a feature, please create an issue to discuss it with the community.
- Use the [Feature Request](https://github.com/Lifely-Finance/Nobile-app/issues/new?template=feature_request.md) template to provide details.

## Submitting Pull Requests
1. Fork the repository and create your branch from `main`.
2. Make your changes and ensure they follow our code standards.
3. Write clear, concise commit messages following our conventions.
4. Push your changes to your fork and open a pull request.
5. Include a description of your changes and reference any related issues in your pull request.

## Code Standards
- Please follow the project's coding style and conventions to maintain consistency throughout the codebase.
- Use tools like linters and formatters to ensure your code adheres to these standards.

## Commit Message Conventions
- Use present tense: "Add feature" instead of "Added feature".
- Use imperatives: "Fix bug" instead of "Fixes a bug".
- Include the issue number when applicable: "Fixes #123".

## Development Setup Instructions
1. Clone the repository:
   ```bash
   git clone https://github.com/Lifely-Finance/Nobile-app.git
   cd Nobile-app
   ```
2. Install the necessary dependencies:
   ```bash
   npm install
   ```
3. Follow the instructions in the project README for further setup.
4. To start the development server, use:
   ```bash
   npm run start
   ```

Thank you for contributing! Together, we can make this project even better!
