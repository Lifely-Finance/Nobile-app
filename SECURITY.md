# SECURITY.md

## Security Policy
Our security policy aims to provide transparency in how we handle security issues within our organization.

## Vulnerability Reporting
If you discover a vulnerability in our application, please report it to us via email at nobile.finance@gmail.com. We will respond promptly and manage the issue securely and efficiently.

## Supported Versions
We support the following versions of our application:
- Version 1.0.0
- Version 1.1.0
- Version 1.2.0

## Security Measures

### Data Protection
Our application uses AES-GCM encryption to ensure data protection. All sensitive data stored locally is encrypted to prevent unauthorized access.

### Code Security
We maintain code security by regularly updating dependencies and conducting thorough code reviews to identify vulnerabilities early.

## Privacy Practices
We are committed to safeguarding our users' privacy and comply with all relevant regulations regarding data protection.

## Known Issues
At present, there are no known security issues. We continuously monitor the application and address issues as they arise.

## User Security Best Practices
Here are some best practices to enhance user security:
- Use strong, unique passwords.
- Enable two-factor authentication (2FA).
- Regularly update your software.

## Vulnerability Disclosure Timeline
We follow a disclosure timeline as follows:
1. Acknowledge receipt of the report within 48 hours.
2. Fix the vulnerability within 30 days, depending on severity.
3. Notify the reporter after the fix is deployed.

For any inquiries, please reach out to nobile.finance@gmail.com
